# AI-DLC Workspace

This folder keeps the working artifacts for building the 13KING landing page with an AI-assisted development lifecycle.

## Flow

1. Charter: define the business goal, audience, constraints, and human decision makers.
2. Requirements: capture user needs, content requirements, compliance checks, and acceptance criteria.
3. Design: shape the information architecture, copy direction, visual direction, and technical approach.
4. Planning: break the work into small tasks that can be implemented and reviewed.
5. Implementation: track code changes, prompt logs, assumptions, and handoff notes.
6. Validation: verify behavior, design quality, accessibility, performance, and SEO.
7. Release: prepare launch notes, rollback notes, and deployment checks.
8. Operations: record analytics, maintenance, experiments, and improvement ideas.

## Working Rule

Each AI-generated output should leave behind:

- The prompt or instruction that produced it.
- The human decision made after reviewing it.
- Any assumptions that still need confirmation.
- The validation result before it is considered done.
